# 데일리 리포트

## 오늘의 목표
- Role 1 로컬 LLM 파이프라인과 번역, 가드레일을 안정화하는 것

## 실제 수행한 작업
- `llama-server` 자동 준비 경로를 추가하고 `Q4_K_S`, `--hf-file`, `--ctx-size`, `max_tokens`, `--reasoning off` 기본값으로 지연을 줄임
- GPU/Metal 기동 실패 시 CPU fallback으로 재시도하고, 다른 모델로 떠 있는 `llama-server`는 재사용하지 않고 명시적으로 실패하게 바꿈
- Role 1 영문 압축을 Kompress 기반으로 바꾸고 PH가 모델 입력에 직접 들어가지 않도록 `ph_masked_input`/`raw_input` 검증을 맞춤
- 번역 fallback/final retry에서 placeholder 보호 규칙을 강화하고 placeholder integrity가 깨진 후보를 버리도록 수정함
- 기본 llama 모델 참조와 관련 문서/테스트를 최신 코드 기준으로 정리함

## 이슈 상황 및 해결 방법
- 이슈: 로컬 모델 기동 실패와 모델 재사용, 그리고 placeholder/리터럴 보호가 느슨하면 번역과 검증 결과가 흔들릴 수 있었음
- 해결 방법: CPU fallback, explicit failure, Kompress 압축, placeholder 우선순위 검증, 모델 기본값 통일로 fallback 품질을 안정화함

## 내일의 계획
- Role 1과 llama.cpp 경로의 실제 입력 사례를 더 넓게 검증함
- 문서에 반영된 기본값과 실행값을 계속 맞춤

## 느낀점
- 로컬 모델 연동은 기능 추가보다 기본값과 검증 규칙을 먼저 고정하는 것이 중요했다.
