# 데일리 리포트

## 오늘의 목표
- task graph 분류, 세션 ID, 실행 경로와 문서 정합성을 함께 다듬는 것

## 실제 수행한 작업
- `TaskGraphProcessor`에 `IDIOM_PATTERNS` pre-check와 질문형 `?$` fallback을 넣어 숙어/질문형 오분류를 수정함
- `document` 후속 작업이 끊기지 않도록 `FLOWS_TO.document`와 `docs/TYPE_DEFINITION.md`를 갱신하고 회귀 테스트를 추가함
- session id를 base62로 만들고 충돌, 로드 실패, REPL continuation `t1` 충돌을 막는 안정화 작업을 적용함
- `createRole2PromptInput()`이 `compressed_prompt`가 아니라 `normalized_input`을 넘기도록 바꾸고 polite prefix 제거를 보강함
- `TaskSentenceSplitter`에서 산술 표현, 순서 표식, 약어/설명성 구문 오분리를 줄이고 `task.type` persistence도 Orchestrator에 반영함
- 관련 unit/integration/build/typecheck 테스트를 함께 묶어 회귀를 확인함

## 이슈 상황 및 해결 방법
- 이슈: 숙어(`make sure`), 문서 후속 작업, 질문형 문장, REPL continuation, normalized/compressed prompt 차이 때문에 분류와 handoff가 서로 어긋날 수 있었음
- 해결 방법: precedence를 좁히고 normalized input을 기준으로 handoff하며, session ID와 graph/test를 함께 보강해 재현 가능성을 낮춤

## 내일의 계획
- role handoff와 adapter 실행 경로의 회귀 테스트를 다시 점검함
- 문서와 코드의 표현 차이를 남은 구간까지 맞춤

## 느낀점
- 실행 경로와 문서 설명을 같이 다듬어야 실제 사용 시 혼선을 줄일 수 있다는 점이 분명했다.
